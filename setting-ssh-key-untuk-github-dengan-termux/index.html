Setting Ssh Key Untuk Github Dengan Termux

Toufiq Nuur on 09 Jan 2021

#git #github #ssh

Ada beberapa cara untuk melakukan koneksi dengan github diantaranya dengan ssh. Mungkin kalian kurang familiar dengan cara ini, karena kebanyakan masih menggunakan https.

Menggunakan metode https memang cukup mudah, tetapi kalian harus memasukkan username dan password setiap kali melakukan push ke github. Ribet bukan? Nah disinilah ssh hadir untuk menutupi kekurangan tersebut.

Ssh memungkinkan kita melakukan push repository tanpa login. Ya tanpa memasukkan username dan password setiap kali melakukan push.

Penasaran yuk simak langkah-langkah setting ssh key untuk github berikut.

1. Menginstal layanan SSH

Pertama kita install dulu layanan sshnya, disini kita akan menggunakan openssh. Kita ketikan perintah berikut untuk menginstal layanan openssh.

pkg install openssh

Setelah selesai kita lanjut ke tahap berikutnya.

2. Membuat key SSH

Untuk membuat key SSH, jalankan perintah ssh-keygen pada termux. Kemudian masukan id (nama) SSH anda misalnya remote-1. Untuk passpharse bisa kita kosongkan.

$ ssh-keygen
Generating public/private rsa key pair.

Enter passphrase (empty for no passphrase):
Enter same passphrase again:
Your identification has been saved in remote-1
Your public key has been saved in remote-1.pub
The key fingerprint is:
SHA256:EEJXWsaeJIjRq/SK/gyPOQRDSdyi0FfQicybrpuXqVo u0_a273@localhost
The key's randomart image is:

|oo+B+*+++        |
|.=o.Bo+=o        |
|+ .. +o+ .       |
|+ . +  .o        |
|.o +    S        |
| .. o            |
|.oEo o           |
|.oO.+            |
|o=*B             |

Setelah itu ketikan perintah ls maka akan akan terlihat ada dua file baru.

$ ls
remote-1  remote-1.pub

3. Jalankan SSH Agent dan Load SSH Key

Untuk menjalankan ssh agent cukup masukan perintah berikut:

ssh-agent /bin/bash

Berikut kita load ssh keynya dengan perintah:

ssh-add nama_key

Disini nama key yang saya buat adalah remote-1 maka perintahnya:

$ ssh-add remote-1
Identity added: remote-1 (u0_a273@localhost)

4. Tambahkan SSH Key ke Github

Sebelumnya kita ambil dulu kunci publik yang telah kita buat tadi dengan perintah:

cat nama_key.pub

Kemudian copy semua teks yang ditampilkan.

$ cat remote-1.pub

Setelah itu buka github lalu masuk ke Settings> SSH and GPG Key

Masukan key yang telah kita salin sebelumnya lalu klik Add SSH Key.

5. Uji Konektivitas

Terakhir kita tinggal menguji konektivitas SSH ke Github dengan perintah:

ssh -T git@github.com

Jika berhasil maka akan muncul tampilan seperti dibawah ini.

$ ssh -T git@github.com
Hi toufiqnuur! You've successfully authenticated, but GitHub does not provide shell access.

Untuk merasakan perbedaannya coba kalian clone repository dan melakukan push ke github.

Nah begitulah cara mensetting SSH Key untuk konektivitas ke Github. Mungkin agak ribet di awal namun akan sangat bermanfaat setelahnya. Kita bisa melakukan push tanpa harus memasukkan username dan password.

Referensi: help.github.com
