Hugo: Markdown render hooks

 

Toufiq Nuur on 10 Mar 2021

#hugo

Belakangan ini menggunakan hugo saya menemukan sedikit kendala.

Yaitu soal render image, karena menggunakan shortcode jadi tidak bisa menggunakan lazyload. Tentu saja ini membuat performa blog menjadi turun.

Namun sekarang tidak lagi, semenjak saya menemukan Markdown Render Hooks.

Apa itu Markdown Render Hooks?

Markdown Render hooks adalah fitur baru semenjak hugo v0.62.0 yang membuat kita dapat membuat template render untuk shortcode. Sebenarnya fitur ini sudah cukup lama mengingat saat ini hugo sudah mencapai versi v0.81.0.

Tapi ini hanya bekerja jika menggunakan goldmark renderer.

Perlu diingat fitur ini hanya support untuk:

  • image
  • link
  • heading (hugo v0.71.0)

Serta hanya bekerja jika menggunakan goldmark renderer.

Cara menggunakan render hooks

Untuk menggunakan render hooks caranya sangat simple, kita hanya perlu membuat template di folder layouts/_default/_markup.

layouts
└── _default
    └── _markup
        ├── render-heading.html
        ├── render-link.html
        └── render-image.html

Contoh penggunaan untuk heading

Buat file render-heading.html didalam folder layouts/_default/_markup.

<h{{ .Level }} class="heading-{{ .Level }}">{{ .Text }}</h{{ .Level }}>

Itu merupakan contoh penggunaan template untuk heading.

…untuk konten markdownnya tinggal ditulis seperti ini.

### Hello

Maka hasil rendernya menjadi

<h3 class="heading-3">Hello</h3>

Contoh penggunaan untuk image

<img src="{{ .Destination | safeURL }}" alt="{{ .Text }}" class="lazy" loading="lazy" />

Untuk dikonten markdownnya tinggal tulis seperti ini.

![Alt](folder/img.jpg)

Sebelum mengenal markdown render hooks saya menulis seperti

<img src="folder/img.jpg" alt="post" class="lazy" loading="lazy" />

…disetiap post yang saya buat.

Nah begitulah kira-kira penggunaan markdown render hooks pada hugo. Untuk lebih lengkapnya kalian bisa cek didokumentasi hugo disini.

Kesimpulan

Markdown render hooks semakin memudahkan kita dalam mengelola konten di hugo.

Dan saya rasa ini merupakan kemajuan yang bagus, kita tunggu update mengejutkan berikutnya.
